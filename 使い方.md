# 使い方（かんたんガイド）

文章を選択して **Ctrl+Alt+T / Ctrl+Alt+B** を押すと、ローカルLLMが補正案を作って
クリップボードに入れ、結果ウィンドウに「原文／生成文」を並べて出すツールです。
**本文は勝手に書き換えません。** 貼り付け（Ctrl+V）は自分で行います。

---

## ✅ 自動起動：設定済み（今すぐ使えます）

Windows ログイン時に自動で常駐するよう設定済みです。**いま既に動いている**ので、
このまま文章を選択して **Ctrl+Alt+T / Ctrl+Alt+B** を押せば使えます。

- 登録場所: スタートアップ
  `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\文章補正ツール.lnk`
- 起動内容: ホットキー(AHK)常駐 ＋ バックエンド（コンソール無し・`pyw`）
- 中身: [autostart.vbs](autostart.vbs)

### 自動起動を止めたい / 解除したい
上記スタートアップの `文章補正ツール.lnk` を**削除**するだけ（次回ログインから起動しません）。
PowerShell なら:
```powershell
Remove-Item "$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\文章補正ツール.lnk"
```

### いま動いているものを止めたい
- バックエンド: タスクマネージャーで `pythonw.exe`（コマンドラインに run_app.py）を終了、
  またはバックエンドウィンドウの「終了」ボタン
- ホットキー: タスクトレイの AutoHotkey アイコンを右クリック →「Exit」

## はじめの準備（最初の1回だけ）

1. **Python**（3.10以上 / tkinter同梱の通常版でOK）をインストール
2. **AutoHotkey v2** をインストール → https://www.autohotkey.com/
3. **Ollama** をインストール → https://ollama.com/
4. モデルを取得（コマンドプロンプトで1回だけ）
   ```
   ollama pull qwen3:8b
   ```
   （この環境では既に `qwen3:8b` が入っているので追加取得は不要です。
   既定モデルは日本語が強い **qwen3:8b**。思考(thinking)は自動でオフにして高速化しています＝
   typoで約6秒。設定画面で他モデルにも切替できます）

---

## 毎回の起動（2回目以降もこれだけ）

### ① Ollama を起動しておく
Ollama アプリを起動するか、コマンドで：
```
ollama serve
```
（すでに常駐していれば何もしなくてOK）

### ② `start.bat` をダブルクリック
このフォルダの **`start.bat`** をダブルクリックすると、

- バックエンド（補正の本体）＝小さなウィンドウが出ます
- ホットキー（AHK）＝タスクトレイに常駐します

がまとめて立ち上がります。

> バックエンドのウィンドウに「接続状態: 接続OK」と緑で出れば準備完了です。
> 「未接続」と赤なら Ollama が起動していません（①に戻る）。

### ③ 使う
1. メモ帳・Slack・ブラウザなど、どこでも文章を**選択**
2. ショートカットを押す
   - **Ctrl+Alt+T** … タイポ修正（誤字・脱字・誤変換だけ直す）
   - **Ctrl+Alt+B** … ビジネス依頼文化（丁寧な依頼文に整える）
   - 押した直後、画面上部に青い「**✍ 補正中…（経過秒）**」が出ます（処理中の合図）
3. 数秒〜十数秒で通知＋結果ウィンドウが出る（結果はクリップボードにコピー済み）
4. 元の場所で **Ctrl+V** で貼り付け
5. 結果ウィンドウで「採用Y / 不採用N」を押すと品質記録になります（任意）

## タスクトレイ常駐

バックエンドはタスクトレイに常駐します（画面右下の通知領域にアイコン）。
- **ウィンドウの[X]で閉じても終了しません**（トレイに最小化＝常駐継続）
- トレイアイコンを**左クリック/ダブルクリック**でウィンドウを再表示
- トレイアイコンを**右クリック**で「表示／無効にする／終了」メニュー
  - 「無効にする」… 一時的にホットキーの補正を止める（再度有効化可）
  - 「終了」… バックエンドを完全終了

## 履歴

ステータスウィンドウの「履歴」ボタンで直近の補正結果（最大10件）を一覧表示。
ダブルクリックで結果ウィンドウを再表示できます。**メモリのみ・終了で破棄**（本文は保存しません）。

## 終わるとき

- **完全終了**: トレイアイコン右クリック →「終了」、またはウィンドウの「終了」ボタン
- ウィンドウの[X]は常駐継続（終了ではない）
- ホットキー(AHK)も止める場合: タスクトレイの AutoHotkey アイコン右クリック →「Exit」

## 手動で起動したいとき（start.bat を使わない場合）

コマンドプロンプト／PowerShell をこのフォルダで開いて：
```
python run_app.py
```
を実行（これがバックエンド）。別途 `ahk\hotkeys.ahk` をダブルクリックでホットキー常駐。

---

## うまく動かないとき

| 症状 | 確認すること |
|---|---|
| 「接続状態: 未接続」と赤い | Ollama が起動しているか（`ollama serve` / Ollamaアプリ） |
| ショートカットを押しても何も起きない | `start.bat`（またはAHK）が起動しているか。タスクトレイ確認 |
| 「取得失敗」通知が出る | 文章を選択してから押しているか。選択せず押すと失敗します |
| 「モデルが見つかりません」 | `ollama pull qwen2.5:7b-instruct-q4_K_M` を実行したか |
| 生成が遅い／重い | 設定画面でより軽いモデル（gemma3:4b 等）に変更 |

## 設定の変え方（モデル・温度・プロンプト・Corpus）

バックエンドのウィンドウの「設定」ボタンから：
- **一般**：Ollamaの場所、使うモデル（候補から選択＋接続テスト）、temperature、最大文字数、
  完了時の自動コピーON/OFF
- **プロンプト**：モードごとの指示文を編集（空欄にすると初期設定に戻る）
- **Corpus**：「採用Y」した文例を貯めて、次回以降の補正に数件差し込む機能（使うほど寄る）

> ショートカットのキー自体を変えたいときは `ahk\hotkeys.ahk` の `^!t` / `^!b` を編集します。
